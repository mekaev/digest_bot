# AI Digest Assistant - MVP Scope

## 1. Короткое описание проекта
AI Digest Assistant - это Telegram-бот и веб-интерфейс, который помогает пользователю быстро получать персональные дайджесты из выбранных открытых Telegram-каналов за нужный период.

Пользователь:
- выбирает интересующие темы;
- выбирает каналы или готовые подборки;
- настраивает периодичность дайджеста;
- получает структурированную сводку в Telegram и в web UI;
- может задавать вопросы по собранному контенту через RAG-ассистента;
- может отправлять голосовые запросы, которые переводятся в текст и обрабатываются системой.

## 2. Проблема
Telegram-каналы перегружают пользователя информацией. Чтобы быть в курсе, приходится тратить много времени на скроллинг, чтение повторяющихся постов и ручную фильтрацию шума.

## 3. Ценность продукта
Продукт экономит время и снижает информационную перегрузку:
- автоматически собирает посты из выбранных источников;
- отбирает наиболее значимые;
- делает короткую, читаемую сводку;
- оставляет ссылки на оригиналы;
- позволяет быстро спросить у ассистента, что было по интересующей теме.

## 4. Основной сценарий MVP
1. Пользователь заходит на landing.
2. Авторизуется.
3. Подключает Telegram-аккаунт.
4. Выбирает темы и поддерживаемые каналы.
5. Настраивает периодичность.
6. Получает дайджест в Telegram.
7. Открывает историю дайджестов в web UI.
8. Спрашивает у RAG-ассистента о материалах за период.

## 5. Что входит в MVP
### Функционал для пользователя
- Telegram-бот с базовыми командами и доставкой дайджестов.
- Landing page.
- Web UI / mini app с настройками.
- Авторизация.
- Каталог поддерживаемых открытых каналов.
- Выбор тем, каналов и периодичности.
- Автоматическая генерация дайджеста.
- История дайджестов.
- RAG-ассистент по собранным постам и прошлым дайджестам.
- Голосовой ввод вопроса с speech-to-text.

### Функционал для администратора
- Добавление и управление каналами.
- Просмотр статуса синхронизации.
- Дашборд использования.
- Воронка продукта.

## 6. Что НЕ входит в MVP
- Полноценная self-serve индексация любых каналов без ограничений.
- Сложная ML-персонализация поведения пользователя.
- Продвинутые рекомендации на основе collaborative filtering.
- Платежи и подписки.
- Мультиязычность.
- Роли и сложная админская система.
- Реалтайм-стриминг контента.

## 7. Решение по пользовательскому добавлению каналов
### Базовое решение для MVP
В MVP источники ограничены списком поддерживаемых открытых каналов.

### Допустимое расширение MVP+
Добавить режим "импорт подборки" через Telegram folder/share-link, но только как удобный способ массово выбрать каналы, которые уже есть в каталоге или могут быть валидированы системой.

### Почему не стоит делать полностью свободный импорт в ядре MVP
- высокий риск на ingestion и стабильность;
- разные каналы могут быть недоступны для выбранного механизма чтения;
- появляется много edge cases с дубликатами, приватными источниками и пустыми подборками;
- это не критично для защиты MVP, но может съесть много времени.

Итог: self-serve import лучше проектировать как опциональный слой поверх каталога, а не как обязательную базу MVP.

## 8. Соответствие 12 критериям
1. **Решает прикладную задачу** - экономия времени на потреблении контента.
2. **Собран с помощью vibe-coding** - процесс разработки ведется с помощью Codex и AI-assisted workflow.
3. **Использует LLM** - ранжирование, суммаризация, RAG.
4. **Имеет интеграцию с ТГ-ботом** - основная доставка контента.
5. **Имеет лендинг** - отдельная публичная страница продукта.
6. **Имеет web UI** - кабинет, история, настройки, чат.
7. **Имеет авторизацию** - вход в систему и привязка Telegram.
8. **Имеет RAG-ассистента** - ответы по собранным постам.
9. **Имеет базу данных** - хранение пользователей, постов, дайджестов, событий.
10. **Имеет voice input + STT** - голосовой запрос в чат.
11. **Имеет dashboard usage** - метрики использования.
12. **Имеет funnel analytics** - ключевые этапы онбординга и активации.

## 10. Success criteria для MVP
- Пользователь может пройти путь от логина до получения первого дайджеста.
- Дайджест приходит по расписанию.
- В дайджесте есть summary и ссылки на исходные посты.
- Пользователь может задать вопрос по контенту и получить ответ с опорой на источники.
- Есть dashboard и funnel.

## 11. Product positioning
Не "парсер всего Telegram", а:

**AI-сервис для персональных дайджестов по curated open Telegram channels.**

Это реалистично, защищаемо и масштабируется в сторону более гибкого user-added sourcing после MVP.
